# Generating realistic article headlines

2019-08-27 testing out machine learning

This article uses clippings from a Norwegian newspaper (some fake, some real), which means that some of these examples might be hard to understand if you're not a native speaker. Don't worry though, I've provided translations into English wherever it was needed!

# Machine learning: what is it good for?

Lately I've been amazed by a lot of strange and impressive things people have done with machine learning. Even though it's become a bit of a hype-train, it's proven to be a very interesting field with plenty of applications. I'm not at all qualified to say anything smart about this topic, but I recently read about a tool called textgenrnn and it seemed like a tool that I might be able to wrap my head around.

In its simplest form, it works like this: a bunch of lines of text (or text-files) is used for training a network, which is then used to generate other lines of text which resemble its source material. The result is usually of varying accuracy depending on the dataset, and what you were trying to mimic. When generating samples from the network, you could also adjust the "temperature" of the samples. It seems like this setting has to do with letting the network "deviate" slightly from what it has learned in order to produce what it sees as "risky" samples.

I copied about 2000 headlines from a local newspaper, and let it run. I started off letting it use the default settings, but after a while I started changing some of them to see what would happen. This mostly consisted of changing the batch size, how long it was trained (to avoid overfitting), and allocating a part of each test-set to validation while training.

The tool seemed to be mostly plug-and-play: my setup was very minimal!

# Results

Even though I probably won't earn myself a literary award by using this tool just yet, it did produce some cool things!

# Default settings, overfitted

Using the default settings caused it to generate some amusing headlines while it was developing, but as it was trained further it started to prefer giving exact copies of headlines from its training set. After being trained for 150 epochs, 64 % of articles at temperature 0.2 were simple copies from the training set. This went slightly better at temperature 1.0, where only 12 % were copies from the training-set. (from a sample size of 100)

# Default settings, trained for a shorter period (15 epochs)

Setting it up like this caused it to generate headlines that were either hit or miss. The number of gibberish-sounding headlines went up, but it also resulted in some headlines that were harder to distinguish from the real ones. (without being exact copies)

Some examples:

Valget på Solvoll – SE VIDEO!
The decision/election at Solvoll - VIEW VIDEO
Temperature: 0.5

Her blir det stengt på Revetal – se bildene! 
These places will be closed in Revetal - images inside!
Temperature: 0.2

Leserinnlegg: Sport i Re – oppdatert!
Reader's letter: Sports in Re - updated! 
Temperature: 0.2

Setting temperatures higher than 0.8 also caused the actual words that made up the samples to stop making sense. Until now the gibberish was caused from a mismatch between words, but in these cases the words also didn't make sense on their own.

En jang på Ridder mål – SE VIDEO
Tre henderynasse for endte i Vestfold: Vestfold!
Sjekk seind med tregerår i Ordåpentel
# Using validation, word-level, and dropout-rates

After re-reading some of the docs it seemed like I could reduce the severity of some of the problems by tweaking some of the settings. In order to emphasize that all the words themselves should make sense, the network was set to train with the "word-level" set to true. In addition to this, adding a dropout-rate and reserving 20 % of the test set for validation would ensure that it didn't just recite existing articles (and instead learned to generate articles that were as accurate as possible, without being the same)

After training it for about 10 epochs (with a steady decline in validation-loss), validation-loss started increasing even though the reported loss was still decreasing. According to this GitHub post this could signal overfitting, so I would stop it there. I had already set in a dropout rate as the post suggested and didn't feel like experimenting further. This was because I believed the "bottleneck" to be the limited size of the dataset, and not using the wrong settings.

Despite this, it gave some cool results!

Leserinnlegg: Re kommune i sentrum 
Reader's letter: Re municipality in focus
Temperature: 0.2

Se bilder fra 17. mai-feiring i Re 2018
Images from "17th of May"-celebration in Re 2018
Temperature: 0.5

Re-legen i Re kommune – se bilder!
The Re-doctor in Re municipality - view images!
Temperature: 0.2

Even though these results seem alright, it should be repeated that in all the cases (except for the massively overfitted one), most of the generated samples were not coherent. An eventual use case for this would probably still require generating a lot of these, and then cherry-picking the best ones out of these. (This is what I did when gathering samples for this post)

Even so, you could probably tweak the settings further for even better results!

# Closing comments (and some amusing samples)

It was cool seeing how the samples picked up some of the general traits from the actual headlines. The most common one was prefacing a headline with "Leserinnlegg: " ("Reader's letter"), which was a common feature of the dataset. Another similarity was appending "- se bilder/video" ("- view images/video") to the end of the headlines, which was also very common. My use of this tool wasn't exactly ground-breaking, but it was still fun to try out!
